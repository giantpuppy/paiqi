# 🎭 排期天菜 (Paiqi)

> 排期的事，交给排期天菜。

排期天菜是一款专为中国音乐剧观众打造的排期管理 App。

市面上的看剧记录工具都是"看完再记"——买完票才想起来要记录。本产品则从**关注一部剧的那一刻起**就把场次信息纳入排期计划。想看的场次一键标记，已买的票根自动归档，看完的数据自动沉淀。从"关注"到"复盘"，全流程一个 App 搞定。

## 🎯 痛点：为什么"排"在中国尤其痛

一个剧女从"知道一部剧要演了"到"最终买票进场"，中间要经历：关注宣发 → 等排期 → 等演员卡司 → 盘同一周有哪些剧 → 对比演员卡司和时间 → 决定买哪场 → 等开票 → 抢票。

现实往往不是"抢到就结束"——热门场次秒售罄，观众转入盘票群和私域二手交易，像做投资一样盯盘、比价、出手，能不能拿到票全靠人脉和运气。

这条路径里，最耗精力的不是"记"，是**"排"**——买票之前横向对比剧目和演员卡司、纵向追踪全周期动向、做出"买哪场"的决策。而且中国音乐剧市场还有几个特殊规则让"排"变得更痛：**开票时演员卡司没出全**，不敢买但好位置不等人；**票不能退，出票很难**，买错了只能自己在二手平台降价转手；**主办方临近打折**，买早怕被背刺买晚怕买不到；**排期信息散落五六个平台**，每次决策都要手动拼凑。

但市面上的工具——售票类的大麦、猫眼聚焦在购票环节，记录类的记录现场、剧在聚焦在看完之后的记录——**没有一个覆盖从"关注"到"排"的前半段路径**。

## 💡 解决方案：排 · 记 · 存

**排**——解决"买哪场"的问题。可视化排期流横向纵向对比演员卡司和时间，想看的场次一键 mark，自动同步到月历总览。

**记**——解决"买了什么"的问题。已购票场次集中管理，票根、座位、待办事项一场场列清楚，不再临场失忆。

**存**——解决"看了多少"的问题。数据自动沉淀，月底年底自动生成可视化图表和看剧报告。

> 看剧的快乐不该被排期吃掉。减轻排期的负担，让精力花在值得沉淀的事情上。

## ✨ 核心功能

### 📸 OCR 智能识别
拍照即识别，3 秒录入一场。支持排期表 / 演员卡司表两种格式，自动解析日期、角色、演员信息，并内置知识库自学习纠错。

### 📅 多视图日历
- **月历视图** — 全月排期一目了然，农历显示
- **双周/周视图** — 快速查看近期安排
- **年历视图** — 全年 12 月网格，状态色块标记
- 支持手势滑动切换视图

### 📊 剧场流排期（甘特图）
- **3日/7日模式** — 双指缩放切换
- 左侧剧目栏 + 右侧时间轴
- 海报卡片封面，同剧多场纵向堆叠
- 今天红色指示线，周末深色背景
- 点击场次快速切换状态

### 🎭 剧目管理
- 添加剧目：表格录入 / OCR 识别 双模式
- 单场详情：时间、座位、票价、卡司
- 月度工作台：按月聚合所有剧目卡片
- 演员选择器：搜索 + 手动新增

### 🖼 个人中心画廊墙
- 观演数据总览：总场次、总花费、票面价、省钱额
- 追踪剧目数、Top 3 演员出场统计
- 设置页独立：OCR 配置、备份恢复、退出登录

### 👤 用户与数据
- 本地账号系统（SHA256+盐）
- 多用户隔离，每用户独立数据库
- JSON 备份/恢复，数据自主可控
- 观剧统计（想看/已买/已观演）

## 🚀 快速开始

### 环境要求
- Flutter SDK ≥3.0.0
- Dart ≥3.0.0
- Android SDK（如运行 Android 端）

### 运行
```bash
# 克隆项目
git clone https://github.com/giantpuppy/leeks-genius.git
cd leeks-genius

# 安装依赖
flutter pub get

# 运行（需连接设备或启动模拟器）
flutter run
```

### 构建 APK
```bash
flutter build apk --release
```
