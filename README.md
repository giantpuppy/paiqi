# 🎭 排期天菜 (LeeksGenius)

> 排期的事，交给排期天菜。

排期天菜是一款专为中国音乐剧观众打造的排期管理 App。

市面上的看剧记录工具都是"看完再记"——买完票才想起来要记录。本产品则从**关注一部剧的那一刻起**就把场次信息纳入排期计划。想看的场次一键标记，已买的票根自动归档，看完的数据自动沉淀。从"关注"到"复盘"，全流程一个 App 搞定。

---

## 🎯 为什么需要排期天菜

一个剧女从"知道一部剧要演了"到"最终买票进场"，中间要经历：关注宣发 → 等排期 → 等演员卡司 → 盘同一周有哪些剧 → 对比演员卡司和时间 → 决定买哪场 → 等开票 → 抢票。

现实往往不是"抢到就结束"——热门场次秒售罄，观众转入盘票群和私域二手交易，像做投资一样盯盘、比价、出手，能不能拿到票全靠人脉和运气。

这条路径里，最耗精力的不是"记"，是**"排"**——买票之前横向对比剧目和演员卡司、纵向追踪全周期动向、做出"买哪场"的决策。而且中国音乐剧市场还有几个特殊规则让"排"变得更痛：

- **开票时演员卡司没出全**，不敢买但好位置不等人
- **票不能退，出票很难**，买错了只能自己在二手平台降价转手
- **主办方临近打折**，买早怕被背刺买晚怕买不到
- **排期信息散落五六个平台**，每次决策都要手动拼凑

但市面上的工具——售票类的大麦、猫眼聚焦在购票环节，记录类的记录现场、剧在聚焦在看完之后的记录——**没有一个覆盖从"关注"到"排"的前半段路径**。

---

## 💡 解决方案：排 · 记 · 存

针对"排"和"记"的脱节，排期天菜围绕三个字构建完整闭环：

**排**——解决"买哪场"的问题。可视化排期流横向纵向对比演员卡司和时间，想看的场次一键 mark，自动同步到月历总览。

**记**——解决"买了什么"的问题。已购票场次集中管理，票根、座位、待办事项一场场列清楚，不再临场失忆。

**存**——解决"看了多少"的问题。数据自动沉淀，月底年底自动生成可视化图表和看剧报告。

> 看剧的快乐不该被排期吃掉。减轻排期的负担，让精力花在值得沉淀的事情上。

## ✨ 核心功能

### 📊 排 · 可视化排期流

- **剧场流排期（甘特图）** — 3日聚焦 / 7日宏观双模式，双指缩放切换
- 海报卡片封面，同剧多场纵向堆叠
- 今天高亮指示线，周末深色背景
- 点击场次快速切换状态（想看/已买/已观演）
- **月度工作台** — 按月聚合剧目卡片，海报网格画廊，支持左右滑切换月份

### 📅 记 · 多视图日历

- **月历视图** — 全月排期一目了然，海报单元格 + 状态色标记，农历显示
- **双周/周视图** — 快速查看近期安排，手势滑动切换
- **年历视图** — 全年 12 月网格，状态色块标记
- 选中日期底部弹出票根列表，点击进入详情

### 🎭 记 · 剧目管理

- 添加剧目：表格录入 / OCR 识别 双模式
- 单场详情：时间、座位、票价、演员卡司
- 场次级排期流：按场次加入/移出排期流，支持批量操作
- 演员选择器：搜索 + 手动新增

### 📸 OCR 智能识别

- 拍照即识别，3 秒录入一场
- 支持排期表 / 演员卡司表两种格式
- 自动解析日期、角色、演员信息
- 内置知识库自学习纠错（Levenshtein 模糊匹配）

### 🖼 存 · 个人中心

- **Hero 指标** — 总场次、追踪剧目数、已购买、已观演
- **剧目场次排序** — 横向条形图，直观对比每部剧看了几场
- **演员排名 / 剧场分布** — 网格展示
- **金额统计** — 实付/票面金额及省钱额
- **月度观演节奏** — 柱状图，点击跳转对应月份

### 👤 用户与数据

- 本地账号系统（SHA256+盐），多用户隔离
- 每用户独立数据库，数据自主可控
- JSON 备份/恢复，支持海报图片导出
- Web 端自动 Demo 模式，扫码即体验

## 🛠 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Flutter 3.x | 跨 Android / iOS / Web / Windows |
| 语言 | Dart ≥3.0.0 | 空安全 |
| UI | Material 3 | 自定义暗黑主题，剧场美学设计 |
| 移动端数据库 | sqflite | SQLite 本地存储，schema v13 |
| Web 端数据库 | sembast + localStorage | 内存数据库 + JSON 持久化 |
| 日历 | table_calendar | 月/双周/周视图 |
| 农历 | lunar | 农历计算 |
| OCR | Google ML Kit / Tesseract.js / 百度 OCR | 移动端/Web端/云端三引擎 |
| 图表 | fl_chart | 个人中心数据可视化 |
| 图片 | image_picker | 相册/拍照 |
| 加密 | crypto (SHA-256) | 密码哈希 |

## 🚀 快速开始

### 环境要求
- Flutter SDK ≥3.0.0
- Dart ≥3.0.0
- Android SDK（如运行 Android 端）

### 运行
```bash
# 克隆项目
git clone https://github.com/giantpuppy/leeks-genius.git
cd leeks-genius

# 安装依赖
flutter pub get

# 运行（需连接设备或启动模拟器）
flutter run
```

### 构建 APK
```bash
flutter build apk --release
```

### 部署 Web
```bash
flutter build web --release --base-href /leeks-genius/
```
